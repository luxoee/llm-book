# 深度审稿 Prompts

这些 prompt 可复制给 AI 审稿人、技术审稿人或编辑，用于商业化发布前复核。

## 1. 商业定位审稿 Prompt

```text
你是一名有技术书商业化出版经验的编辑。请审阅这本书的 README.md、SUMMARY.md 和第一章，判断它是否具备清晰的市场定位。

重点检查：
1. 目标读者是否明确；
2. 读者痛点是否真实；
3. 书的差异化卖点是否足够强；
4. 目录是否能支撑销售页承诺；
5. 是否有内容过窄、过深、过散或不利于商业化表达的问题。

请输出：
- 3 个最强卖点；
- 3 个最大商业化风险；
- 目录调整建议；
- 首页文案修改建议；
- 是否达到可发布状态，按 1–10 打分。
```

## 2. 技术准确性审稿 Prompt

```text
你是一名 LLM inference / vLLM / GPU serving 技术审稿人。请审阅 chapters/ 下所有章节，重点检查 Qwen/Qwen3.6-35B-A3B 与 vLLM v0.20.1 相关事实是否准确。

固定边界：
- 模型：Qwen/Qwen3.6-35B-A3B；
- Serving 框架：vLLM v0.20.1；
- 硬件视角：NVIDIA GPU。

重点检查：
1. 是否把 Qwen3.6 错误简化成普通 Transformer；
2. Gated DeltaNet、Gated Attention、MoE、MTP 的表述是否准确；
3. KV cache、PagedAttention、continuous batching、prefix caching 的因果关系是否清楚；
4. vLLM 源码路径是否与 v0.20.1 对齐；
5. 部署命令和工程建议是否可能误导读者。

请输出：
- 必须修正的问题；
- 建议修正的问题；
- 需要来源确认的问题；
- 可以保留但建议弱化的推断；
- 总体技术可信度评分。
```

## 3. 章节结构与阅读体验审稿 Prompt

```text
你是一名中文技术写作教练。请审阅 chapters/ 下所有章节的阅读体验。

重点检查：
1. 章节顺序是否符合学习曲线；
2. 每章开头是否能说明本章要解决的问题；
3. 生活类比、数学直觉、张量形状、计算流程、显存布局、源码映射、GPU 视角是否衔接自然；
4. 是否存在重复、跳跃、术语未解释、公式过密或工程建议不落地的问题；
5. 每章结尾的总结、公式、源码入口、作业、自测题是否有帮助。

请输出：
- 最需要重写的 5 个位置；
- 最有商业价值的 5 个位置；
- 读者最可能卡住的 5 个位置；
- 建议新增的图表或示意图；
- 是否适合 GitBook 连续阅读。
```

## 4. GitBook 发布检查 Prompt

```text
你是一名 GitBook 发布工程师。请检查这个仓库是否适合直接发布到 GitBook。

重点检查：
1. README.md 是否适合作为首页；
2. SUMMARY.md 是否完整且链接有效；
3. Markdown 标题层级是否规范；
4. 表格、代码块、数学公式是否可能在 GitBook 中渲染异常；
5. raw/、publishing/、chapters/ 的关系是否清楚；
6. 是否有不应出现在正式发布中的内部文件或草稿痕迹。

请输出：
- 阻塞发布的问题；
- 非阻塞但建议修复的问题；
- GitBook 目录结构建议；
- 发布前最终检查命令。
```

## 5. 目标市场匹配 Prompt

```text
你是一名 AI 基础设施课程产品经理。请从市场需求角度评估这本书是否适合作为付费 GitBook / 训练营教材 / 企业内训材料。

请重点评估：
1. 当前 LLM inference、vLLM、MoE、长上下文、GPU serving 的市场需求；
2. 本书主题是否足够聚焦；
3. 读者是否愿意为这类内容付费；
4. 哪些章节最适合做试听或营销文章；
5. 哪些章节需要补案例、图、实验数据才能提升转化。

请输出：
- 最适合的商业形态；
- 建议定价层级；
- 推荐营销卖点；
- 内容短板；
- 上线前优先补强清单。
```
